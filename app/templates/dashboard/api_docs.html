INTERFACE_PROTOCOLS

Overview

The SENTINALL API allows you to programmatically archive content and generate secure redirection tokens. All requests must be authenticated using your unique API_KEY.

Base Endpoint

{{ app_url }}

Authentication

Include your API key in all requests as a query parameter or within the JSON body.

GET {{ app_url }}/api?api=YOUR_API_KEY&url=...

POST {{ app_url }}/api/create

{ "api": "YOUR_API_KEY", "url": "..." }
POST / GET

Create Archive Token

Parameter Type Required Description
api string YES Your unique operator API key
url string YES The destination URL to be archived

Request Example (cURL)

# Using POST with JSON body curl -X POST {{ app_url }}/api/create \ -H "Content-Type: application/json" \ -d '{ "api": "{{ user.api_key }}", "url": "https://example.com/target-content" }'

Response Example (200 OK)

{ "status": "success", "shortenedUrl": "{{ app_url }}/start/gAAAAABp...", "original_url": "https://example.com/target-content" }

Response Codes

200 OK Protocol successful, token returned.
401 ERR Invalid or missing API credentials.
400 ERR Missing parameters or shortener failure.
